# Security policy

Do not open a public issue containing a vulnerability, personal data, credentials, tenant identifiers, or production logs.

Until the repository is transferred to the company organization, report security issues through the company's approved private security/IT incident channel and identify the Qbook repository and affected environment. After transfer, the GitHub organization owner must enable Private Vulnerability Reporting and update this file with the company security contact and response targets.

If credentials may be exposed, revoke and rotate them before code/history cleanup. Preserve only redacted evidence in the incident ticket.
